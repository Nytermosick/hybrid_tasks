# Описание основных файлов

Запуск осуществлять из папки `g1_tasks` из docker-контейнера ilab_docker

## 0. `deploy/mujoco/unitree_mujoco.py`

Стандартный файл запуска симуляции в симуляторе Mujoco через DDS. Параметры симуляции можно настроить через файл `deploy/mujoco/config.py`.

#### ВАЖНО: для корректной работы симуляции нужно оставлять DOMAIN_ID=1, INTERFACE='lo' и ENABLE_ELASTIC_BAND = True. Такие параметры как USE_JOYSTICK, JOYSTICK_TYPE, JOYSTICK_DEVICE на работу никак не влияют (остатки от оригинального кода).

### Аргументы

| Аргумент | Тип | Default | Описание |
|---------|------|----------|----------|
| `--air_fix` | - | `store_true` | Фиксация робота в воздухе для тестов. |

## 1. `deploy/QP_controller.py`

Главный гибридный контроллер, реализующий управление по архитектуре RAMBO.
- Политика на выходе выдаёт вектор из 35 значений - 29 для коррекции joint positions, 6 для коррекции ускорения базы
- QP-задача оптимизации находит оптимальные ground reaction forces (grf) для stance фазы, которые затем преобразуются в моменты для моторов: $\tau = -J^T f$
- Swing Controller формирует траектории для ног в swing фазе через Bezier-траектории и обратную кинематику с использованием псевдообратного якобиана: $\Delta{q} = J^{+} \Delta{x}$

## 2. `deploy/robot_env.py`

Файл, содержащий два класса `ObsData` и `G1_Env`.

### `ObsData`

Объект данного класса представляет собой структуру, содержащую всю необходимую информацию для работы контроллера. Здесь формируется необходимый state для контроллера робота и вектор наблюдений, который затем используется как вход в нейронную сеть.

### `G1_Env`

Данный класс используется непосредственно для связи контроллера и робота Unitree G1 по DDS. Алгоритм работы:
- Инициирует работу DDS. Если запуск происходит на реальном роботе, выключает внутренний контроллер
- Получает первое сообщение `lowstate` от робота. Если робот находится в недопустимом положении, завершает выполнение программы
- Переводит робота в начальную позицию, затем удерживает его в ней до включения контроллера
- В цикле:
  - Получает сообщение `lowstate` от робота. Если робот находится в недопустимом положении, включает `damping mode` и завершает программу
  - Формирует необходимую информацию с использованием Pinocchio модели робота, заполняет структуру класса `ObsData`
  - Отправляет структуру `ObsData` контроллеру, получает управляющие команды, заполняет структуру `lowcmd` и отправляет роботу
  - По команде оператора осуществляет немедленный переход в `damping mode` с последующим завершением программы

## 3. `deploy/main.py` (Основной файл запуска)

Основной файл запуска системы. Управляет режимами работы и главным циклом управления.
- Обрабатывает ввод с клавиатуры / геймпада
- Запускает control loop
- Переключает режимы:
  - Default position
  - QP_controller
  - Damping mode

Для запуска симуляции в mujoco предварительно запустите файл `deploy/mujoco/unitree_mujoco.py`, затем запустите `deploy/main.py`:
```bash
python3 deploy/mujoco/unitree_mujoco.py
python3 deploy/main.py
```

Для запуска контроллера на реальном роботе:
1) Осуществите подключение через Ethernet-кабель;
2) Задайте статический ip-адрес с G1 в одной сети (ip-адрес G1: 192.168.123.161). Проверить подключение можно с помощью команды `ping`;
3) Узнайте название интерфейса подключения Ethernet с помощью команды `ip a` (скорее всего будет `enp3s0`);
4) Запустите `deploy/main.py` с указанием названия интерфейса (см. список аргументов):
    ```bash
    python3 deploy/main.py --interface enp3s0
    ```

### Опционально: Можно использовать джойстик для управления роботом. Для этого:
1) Подключите джойстик через провод или bluetooth;
2) Запустите `deploy/main.py` с аргументом `joystick_on` (см. список аргументов):
    ```bash
    python3 deploy/main.py --interface enp3s0 --joystick_on
    ```

### Аргументы
Если запускаете с аргументами, то нужно писать их имена, затем значения:

| Аргумент | Тип | Default | Описание |
|---------|------|----------|----------|
| `--interface` | str | `"lo"` | Имя сетевого интерфейса (eth0, wlan0, enp3s0). С аргументом по-умолчанию запускается контроллер в симуляции. |
| `--air_fix` | - | `store_true` | Фиксация робота в воздухе для тестов. |
| `--joystick_on` | - | `store_true` | Включает поддержку джойстика, если аргумент указан. |
| `--policy_path` | str | `"deploy/policies/05/exported/policy.pt"` | Путь до файла политики. |
